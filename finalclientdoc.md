## Описание проекта

Наша команда разработала современное клиентское веб-приложение (**Приложение А**) для абитуриентов и посетителей филиала Томского индустриально-гуманитарного колледжа «Anmicius». Приложение предоставляет удобный интерфейс для взаимодействия с информацией о колледже и интегрируется с разработанным нами REST API (**Приложение Б**), которое выступает в роли бэкенда и было представлено в отдельной документации.

### Целевая аудитория

- **Абитуриенты** — получение информации о специальностях, прохождение профориентационного теста
- **Студенты** — доступ к новостям, документам, FAQ
- **Родители** — ознакомление с условиями приёма, важными датами
- **Посетители** — общая информация о колледже, галерея, контакты

## Проблема и решение

### Проблема (до внедрения)

| Проблема | Последствия |
|----------|-------------|
| Информация распространяется через бумажные буклеты | Высокие затраты на печать, устаревшие данные |
| Нет единого источника информации | Противоречивые данные из разных источников |
| Абитуриенты не могут дистанционно получить рекомендации | Низкая конверсия абитуриентов |
| Обновление информации занимает дни | Задержки в публикации важных новостей |
| Отсутствие мобильной версии | Потеря аудитории смартфонов |

### Наше решение

Мы создали **единое клиентское приложение**, которое:

✅ **Централизует всю информацию** о колледже в одном месте  
✅ **Предоставляет интерактивный профориентационный тест** с персональными рекомендациями  
✅ **Обеспечивает мгновенное обновление данных** через интеграцию с нашим API  
✅ **Адаптировано под все устройства** (desktop, tablet, mobile)  
✅ **Работает быстро** благодаря кэшированию и оптимизации

## Интеграция с API (Приложение Б)

### Endpoints, которые мы используем

Наше клиентское приложение взаимодействует с разработанным нами REST API (Приложение Б, проект `tpgk_server`). Ниже приведены основные endpoints:

| Endpoint | Метод | Описание | Использование в приложении |
|----------|-------|----------|---------------------------|
| `/api/v1/about` | GET | Информация о колледже | Главная страница |
| `/api/v1/admission` | GET | Приёмная кампания | Раздел "Поступление" |
| `/api/v1/specialties` | GET | Список специальностей | Каталог специальностей |
| `/api/v1/specialties/{code}` | GET | Детали специальности | Страница специальности |
| `/api/v1/specialties/{code}/facts` | GET | Факты специальности | Карточка специальности |
| `/api/v1/news` | GET | Список новостей | Лента новостей |
| `/api/v1/news/{slug}` | GET | Детали новости | Страница новости |
| `/api/v1/faq` | GET | Часто задаваемые вопросы | Раздел FAQ |
| `/api/v1/documents` | GET | Документы | Библиотека документов |
| `/api/v1/images` | GET | Галерея изображений | Фотогалерея |
| `/api/v1/test/questions` | GET | Вопросы теста | Страница тестирования |
| `/api/v1/test/results` | POST | Отправка результатов | Обработка результатов теста |

### Пример запроса к API

```typescript
// Получение списка специальностей
async function fetchSpecialties(page = 1, filters = {}) {
  const params = new URLSearchParams({
    page: page.toString(),
    limit: '10',
    ...filters
  });
  
  const response = await fetch(`https://api.anmicius.ru/api/v1/specialties?${params}`);
  const data = await response.json();
  
  return {
    specialties: data.items,
    total: data.total,
    currentPage: data.page,
    totalPages: Math.ceil(data.total / data.limit)
  };
}

// Отправка результатов теста
async function submitTestResults(answers) {
  const response = await fetch('https://api.anmicius.ru/api/v1/test/results', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ answers })
  });
  
  return await response.json(); // Возвращает рекомендации специальностей
}
```

### Обработка ошибок

Мы реализовали надёжную систему обработки ошибок для обеспечения стабильной работы:

```typescript
class ApiClient {
  async request(endpoint, options = {}) {
    try {
      const response = await fetch(endpoint, {
        ...options,
        headers: {
          'Content-Type': 'application/json',
          ...options.headers
        }
      });
      
      if (!response.ok) {
        const error = await response.json();
        throw new ApiError(error.detail, response.status);
      }
      
      return await response.json();
    } catch (error) {
      if (error instanceof ApiError) {
        this.handleError(error);
      } else {
        // Сетевая ошибка
        this.showOfflineMessage();
      }
      throw error;
    }
  }
}
```

### Кэширование на клиенте

Для улучшения производительности и снижения нагрузки на API мы внедрили многоуровневое кэширование:

1. **HTTP Cache** — использование заголовков Cache-Control от API
2. **LocalStorage** — сохранение редко изменяемых данных (справочники, FAQ)
3. **In-Memory Cache** — кэш в памяти для текущей сессии

```typescript
class CacheService {
  private cache = new Map();
  
  async get(key, ttl = 300000) {
    const item = this.cache.get(key);
    if (!item) return null;
    
    if (Date.now() > item.expiry) {
      this.cache.delete(key);
      return null;
    }
    
    return item.data;
  }
  
  async set(key, data, ttl = 300000) {
    this.cache.set(key, {
      data,
      expiry: Date.now() + ttl
    });
  }
}
```
