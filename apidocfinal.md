# Приложение Б: Серверная часть и API системы профориентации (tpgk_server)

## Введение

Наша команда разработала высокопроизводительное серверное приложение **tpgk_server** (Приложение Б), которое выступает в качестве центрального API для всей системы профориентации и управления приёмной кампанией. Данное решение обеспечивает надежную обработку данных, бизнес-логику и безопасный доступ к информации для клиентских приложений (включая Приложение А), мобильных клиентов и внешних интеграций.

## Описание проекта

Мы создали масштабируемый бэкенд на базе современного стека технологий, ориентированный на высокую нагрузку и безопасность. Приложение Б реализует всю ключевую функциональность системы: управление пользователями, проведение тестирования, каталог специальностей, мониторинг приёмной кампании и аналитику.

## Архитектура решения

Для обеспечения чистоты кода, легкой поддерживаемости и тестируемости мы применили архитектурный паттерн **Clean Architecture**. Система разделена на четкие слои:

1.  **Domain (Домен):** Содержит бизнес-модели и интерфейсы репозиториев. Не зависит от фреймворков и баз данных.
2.  **Application (Приложение):** Реализует бизнес-правила и сценарии использования (Use Cases). Оркестрирует поток данных между доменом и инфраструктурой.
3.  **Infrastructure (Инфраструктура):** Содержит реализации работы с базами данных (SQLAlchemy/Alembic), кэшем (Redis), файловыми хранилищами (MinIO) и внешними сервисами.
4.  **Presentation (Презентация):** Слой API (FastAPI), обрабатывающий HTTP-запросы, валидацию данных (Pydantic) и аутентификацию.

```
[Клиенты (Web, Mobile)] 
       ⬇️ HTTPS
[API Gateway / Load Balancer]
       ⬇️
[Приложение Б: tpgk_server (FastAPI)]
   ├── Presentation (Routes, Schemas)
   ├── Application (Services, DTOs)
   ├── Domain (Entities, Repos Interfaces)
   └── Infrastructure (DB, Cache, Storage)
```

## Ключевые возможности и реализованные функции

В ходе разработки мы внедрили следующий функционал:

*   **Система аутентификации и авторизации:** Реализована на основе JWT-токенов с поддержкой ролевой модели (абитуриент, администратор, модератор).
*   **Профориентационное тестирование:** Полный цикл проведения тестов, подсчет результатов, алгоритмы рекомендаций специальностей на основе ответов.
*   **Управление специальностями:** CRUD-операции для каталога направлений подготовки, включая проходные баллы и количество мест.
*   **Приёмная кампания:** Механизмы подачи заявок, отслеживание статуса документов и формирование рейтинговых списков.
*   **Файловое хранилище:** Интеграция с MinIO для безопасного хранения сканов документов и аватаров пользователей.
*   **Миграции базы данных:** Настроена система версионирования схемы БД с помощью Alembic.

## Технологический стек

Мы выбрали следующие технологии для достижения максимальной производительности и надежности:

*   **Язык:** Python 3.10+
*   **Фреймворк:** FastAPI (асинхронная обработка запросов, автоматическая генерация Swagger/OpenAPI документации).
*   **База данных:** PostgreSQL (основное хранилище).
*   **ORM:** SQLAlchemy (асинхронный режим).
*   **Валидация:** Pydantic V2.
*   **Кэширование:** Redis.
*   **Контейнеризация:** Docker & Docker Compose для упрощения развертывания.

## Интерфейсы взаимодействия (API)

Приложение Б предоставляет RESTful API с автоматической документацией. Основные группы эндпоинтов:

| Группа | Префикс | Описание |
|--------|---------|----------|
| **Auth** | `/api/v1/auth` | Регистрация, логин, обновление токенов |
| **Tests** | `/api/v1/tests` | Получение тестов, отправка ответов, результаты |
| **Specialties** | `/api/v1/specialties` | Каталог специальностей, фильтрация, детали |
| **Applications** | `/api/v1/applications` | Подача заявлений, статусы, документы |
| **Admin** | `/api/v1/admin` | Панель управления для модерации данных |

### Пример запроса к API

Получение списка рекомендованных специальностей после прохождения теста:

```http
GET /api/v1/tests/recommendations?user_id=123
Authorization: Bearer <JWT_TOKEN>
```

Ответ сервера (JSON):
```json
{
  "status": "success",
  "data": [
    {
      "specialty_code": "09.03.04",
      "name": "Программная инженерия",
      "match_score": 95,
      "budget_slots": 50
    },
    {
      "specialty_code": "09.03.02",
      "name": "Информационные системы",
      "match_score": 88,
      "budget_slots": 30
    }
  ]
}
```

## Безопасность и надежность

Особое внимание мы уделили безопасности данных:
*   Все пароли хранятся в хешированном виде (bcrypt).
*   Реализована защита от SQL-инъекций и XSS-атак благодаря особенностям фреймворка и валидации входных данных.
*   Внедрено логирование критических событий и ошибок для оперативного мониторинга.
*   Настроено резервное копирование базы данных.
