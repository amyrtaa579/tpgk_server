# Публичные эндпоинты API

Все эндпоинты префикса `/api/v1` доступны **без аутентификации** и кэшируются в Redis.

## Базовый URL

```
https://api.anmicius.ru/api/v1
```

## GET /api/v1/specialties

Список специальностей с пагинацией и фильтрами.

**Кэш:** 5 минут

**Параметры запроса:**

| Параметр | Тип | Обязательный | По умолчанию | Описание |
|----------|-----|-------------|-------------|----------|
| `page` | int | Нет | 1 | Номер страницы |
| `limit` | int | Нет | 10 | Элементов на странице (1-50) |
| `search` | string | Нет | — | Поиск по названию/коду |
| `form` | string | Нет | — | Фильтр: `"budget"` или `"paid"` |
| `popular` | bool | Нет | false | Только популярные специальности |

**Запрос:**
```
GET /api/v1/specialties?page=1&limit=10&search=программ
```

**Ответ (200 OK):**
```json
{
  "items": [
    {
      "id": 1,
      "code": "09.02.07",
      "name": "Информационные системы и программирование",
      "short_description": "Подготовка специалистов в области разработки ПО..."
    },
    {
      "id": 2,
      "code": "15.02.19",
      "name": "Технология аdditивного производства",
      "short_description": "Специалисты по 3D-печати..."
    }
  ],
  "total": 12,
  "page": 1,
  "limit": 10,
  "pages": 2
}
```

**Фильтр `form`:**
- `budget` — только специальности с бюджетными местами
- `paid` — только специальности с платными местами

**Параметр `popular`:**
- При `popular=true` возвращает специальности, у которых есть интересные факты

## POST /api/v1/test/results

Отправить ответы профориентационного теста и получить рекомендацию.

**Требование:** Ровно **10** ответов.

**Тело запроса:**
```json
{
  "answers": [
    {"question_id": 1, "selected": 0},
    {"question_id": 2, "selected": 1},
    {"question_id": 3, "selected": 2},
    {"question_id": 4, "selected": 3},
    {"question_id": 5, "selected": 0},
    {"question_id": 6, "selected": 1},
    {"question_id": 7, "selected": 2},
    {"question_id": 8, "selected": 0},
    {"question_id": 9, "selected": 3},
    {"question_id": 10, "selected": 1}
  ]
}
```

**Ответ (200 OK):**
```json
{
  "recommendation": "На основе ваших ответов, рекомендуем специальность:",
  "motivation": "Вы проявляете интерес к техническим дисциплинам и имеете склонность к аналитическому мышлению.",
  "recommended_specialties": [
    {
      "code": "15.02.19",
      "name": "Технология аддитивного производства",
      "score": 15
    },
    {
      "code": "21.02.03",
      "name": "Монтаж и техническая эксплуатация оборудования",
      "score": 12
    },
    {
      "code": "09.02.07",
      "name": "Информационные системы и программирование",
      "score": 8
    }
  ]
}
```

### Алгоритм расчёта рекомендации

1. Для каждого ответа берётся `answer_scores[selected_index]` соответствующего вопроса
2. Баллы суммируются по кодам специальностей
3. Топ-3 специальности с наибольшим баллом возвращаются
4. Если совпадений нет — возвращаются специальности по умолчанию (технические)

**Ошибка валидации (не 10 ответов):**
```json
{
  "detail": "Необходимо ответить ровно на 10 вопросов. Получено: 9"
}
```

## Health Check

### GET /health

Проверка состояния всех компонентов системы.

**Запрос:**
```
GET /health
```

**Ответ (200 OK):**
```json
{
  "status": "ok",
  "version": "1.0.0",
  "services": {
    "database": {"status": "ok"},
    "redis": {"status": "ok"},
    "minio": {"status": "ok"}
  }
}
```

**Ответ при деградации (200 OK, status = "degraded"):**
```json
{
  "status": "degraded",
  "version": "1.0.0",
  "services": {
    "database": {"status": "ok"},
    "redis": {"status": "error", "message": "Connection refused"},
    "minio": {"status": "unavailable", "message": "MinIO not configured"}
  }
}
```

> **Примечание:** Эндпоинт всегда возвращает HTTP 200. Статус системы определяется полем `status`: `"ok"` (все компоненты работают) или `"degraded"` (часть компонентов недоступна).
